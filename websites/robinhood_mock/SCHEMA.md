# robinhood_mock Schema

**Deploy order**: 41 (alphabetical among all *_mock dirs, BASE_PORT=8000 → port 8041)
**Base URL**: `http://172.17.46.46:8041/`
**Go Endpoint**: `GET /go?sid=<sid>` → `{initial_state, current_state, state_diff}`
**Inject**: `POST /post?sid=<sid>` with body `{"action":"set","state":{...}}`
**Update current only**: `POST /post?sid=<sid>` with body `{"action":"set_current","state":{...}}`
**Reset**: `POST /post?sid=<sid>` with body `{"action":"reset"}`
**State read**: `GET /state?sid=<sid>` → `{stored_state, has_custom_state, sid}`
**File upload**: `POST /upload?sid=<sid>` (multipart/form-data) → `{success, files: [{original_name, stored_name, size, content_type, url}]}`
**File serve**: `GET /files/<sid>/<filename>`

### Transaction

Represents both trade orders and cash transfers. Discriminated by the `type` field.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | string | generated | Unique transaction ID (e.g. `"txn_001"`, `"txn_${Date.now()}"`) |
| `date` | string | now ISO | ISO 8601 timestamp of the transaction. |
| `symbol` | string | `""` | Stock ticker symbol. Empty string `""` for transfer transactions. |
| `name` | string | `""` | Company name for trades; `"Deposit"` or `"Withdrawal"` for transfers. |
| `type` | string | `"market"` | Transaction type: `"market"`, `"limit"`, `"stop"`, or `"transfer"`. |
| `side` | string | `"buy"` | Direction: `"buy"` or `"sell"`. For transfers: `"buy"` = deposit, `"sell"` = withdrawal. |
| `quantity` | number | `0` | Number of shares. `0` for transfer transactions. |
| `price` | number | `0` | Execution price per share. `0` for transfer transactions. |
| `totalAmount` | number | `0` | Total value: `quantity * price` for trades; dollar amount for transfers. |
| `status` | string | `"filled"` | Order status: `"filled"`, `"cancelled"`, or `"pending"`. Transfers are always `"filled"`. |
| `limitPrice` | number\|null | `null` | Limit price for limit orders; `null` otherwise. |
| `stopPrice` | number\|null | `null` | Stop price for stop orders; `null` otherwise. |

**Pending orders** (limit or stop type): created without immediately modifying portfolio. Buying power is reserved for pending buy orders. Orders remain `"pending"` until cancelled (status → `"cancelled"`) or filled externally.

## Store Actions (Client-Side)

These functions are exposed via React context (`useStore()`). All mutations update `localStorage` and trigger a re-render.

| Action | Signature | Description |
|--------|-----------|-------------|
| `placeOrder` | `(symbol, type, side, quantity, price, limitPrice?, stopPrice?)` | Core order placement. **Market orders** (`type="market"`) fill immediately: modifies `portfolio`, `user.cashBalance`, `user.buyingPower`, appends filled transaction. **Limit/stop orders** (`type="limit"` or `"stop"`) create a `"pending"` transaction; only reserves `buyingPower` for buy side; does NOT modify `portfolio`. Throws `"Insufficient buying power"` or `"Insufficient shares"` on validation failure. |
| `placeLimitOrder` | `(symbol, side, quantity, limitPrice)` | Convenience wrapper for `placeOrder(..., "limit", ...)`. |
| `placeStopOrder` | `(symbol, side, quantity, stopPrice)` | Convenience wrapper for `placeOrder(..., "stop", ...)`. |
| `toggleWatchlist` | `(symbol)` | Adds symbol to `watchlist` if absent; removes it if present. |
| `addToWatchlist` | `(symbol)` | Adds symbol to `watchlist` if not already present (no-op if duplicate). |
| `removeFromWatchlist` | `(symbol)` | Removes symbol from `watchlist`. |
| `cancelOrder` | `(orderId)` | Finds transaction by ID. If status is `"pending"`, sets status to `"cancelled"`. For pending buy orders, also restores `user.buyingPower` by the order's `totalAmount`. No effect if already filled or cancelled. |
| `markAlertRead` | `(alertId)` | Sets `alerts[i].read = true` for the matching alert ID. |
| `markAllAlertsRead` | `()` | Sets `read = true` on all alerts. |
| `updateSettings` | `(settingsUpdate: object)` | Shallow-merges `settingsUpdate` into `state.settings`. Used by Account page for `dividendReinvestment`, `orderNotifications`, and `theme`. |
| `addTransfer` | `(type: "deposit"\|"withdraw", amount: number)` | Cash transfer. **Deposit**: adds `amount` to `user.cashBalance` and `user.buyingPower`; appends a `transfer` transaction with `side="buy"`, `name="Deposit"`. **Withdraw**: subtracts `amount` from `user.cashBalance` and `user.buyingPower` (no-op silently if `cashBalance < amount`); appends a `transfer` transaction with `side="sell"`, `name="Withdrawal"`. All transfer transactions have `type="transfer"`, `status="filled"`, `quantity=0`, `price=0`, `symbol=""`. |
| `getGoData` | `()` | Returns `{initial_state, current_state, state_diff}` for the `/go` debug page. |

### Price Simulation (Automatic)

Every **3 seconds**, an interval updates all stock prices:
- Each stock's `currentPrice` shifts by a random ±0.2% factor.
- `change` = `newPrice - prevClose`
- `changePercent` = `(change / prevClose) * 100`
- `user.portfolioValue` is recalculated as the sum of `(holding.quantity * stock.currentPrice)` for all portfolio holdings.

## API Endpoints (Vite Middleware)

These endpoints are provided by `vite.config.js` and work identically in both `dev` and `preview` modes.

### POST /post?sid=\<sid\>

Set or reset server-side state. The client fetches this on first load when a `sid` query parameter is present.

**Body `{"action":"set","state":{...}}`**
- Writes `state` to `.mock-states/<sid>.json` (or `.mock-state.json` for no-sid).
- Also writes `.mock-states/<sid>.initial.json` on first set (used for `state_diff` baseline).
- Optional `"merge": true` to deep-merge into existing state instead of replacing.
- Response: `{"success":true,"sid":"...","message":"State updated.","state":{...}}`

**Body `{"action":"set_current","state":{...}}`**
- Updates ONLY the current state file (`.json`). Does NOT touch the `.initial.json` baseline.
- Used by `golden_patch.py` to simulate correct task completion without disturbing the initial state.
- Optional `"merge": true` for deep-merge.
- Response: `{"success":true,"message":"Current state updated. Initial state preserved.","state":{...}}`

**Body `{"action":"reset"}`**
- Deletes the state file so the app reverts to default on next load.
- Response: `{"success":true,"sid":"...","message":"State reset to default."}`

### GET /state?sid=\<sid\>

Returns the raw stored state (written by `/post`).
Response: `{"stored_state":{...}|null,"has_custom_state":true|false,"sid":"..."}`

### GET /go?sid=\<sid\>

Returns state diff for RL training evaluation. Reads `.initial.json` and `.json` files from disk.
Response: `{"initial_state":{...},"current_state":{...},"state_diff":{...}}`

The diff format uses per-key `{"modified": <value>}` or `{"added": <value>}` entries for changed keys.

### POST /upload?sid=\<sid\>

Upload files (multipart/form-data). Stores files in `.mock-files/<sid>/`.
Response: `{"success":true,"files":[{"original_name":"...","stored_name":"...","size":N,"content_type":"...","url":"/files/<sid>/<stored_name>"}]}`

### GET /files/\<sid\>/\<filename\>

Serve a previously uploaded file. Returns file bytes with appropriate `Content-Type`.

## Observable State Changes (for RL Evaluation)

| User Action | Changed State Fields |
|-------------|----------------------|
| Buy stock — market order | `user.cashBalance` decreases; `user.buyingPower` decreases; `portfolio[symbol].quantity` increases (entry created if new); `portfolio[symbol].avgPrice` recalculated (weighted avg); `transactions` gains 1 entry (`type="market"`, `status="filled"`); `user.portfolioValue` recalculated |
| Sell stock — market order | `user.cashBalance` increases; `user.buyingPower` increases; `portfolio[symbol].quantity` decreases (entry deleted if reaches 0); `transactions` gains 1 entry (`type="market"`, `status="filled"`); `user.portfolioValue` recalculated |
| Buy stock — limit order | `user.buyingPower` decreases (reserved); `portfolio` unchanged; `transactions` gains 1 entry (`type="limit"`, `status="pending"`, `limitPrice` set) |
| Sell stock — limit order | `portfolio` unchanged; `user.buyingPower` unchanged; `transactions` gains 1 entry (`type="limit"`, `status="pending"`, `limitPrice` set) |
| Buy/sell stock — stop order | Same as limit order but `type="stop"`, `stopPrice` set instead of `limitPrice` |
| Cancel pending order | `transactions[i].status` changes `"pending"` → `"cancelled"`; for pending buy orders: `user.buyingPower` increases by `transactions[i].totalAmount` |
| Deposit (from Transfers or History modal) | `user.cashBalance` increases by amount; `user.buyingPower` increases by amount; `transactions` gains 1 entry (`type="transfer"`, `side="buy"`, `name="Deposit"`, `status="filled"`, `quantity=0`, `price=0`) |
| Withdraw (from Transfers or History modal) | `user.cashBalance` decreases by amount; `user.buyingPower` decreases by amount; `transactions` gains 1 entry (`type="transfer"`, `side="sell"`, `name="Withdrawal"`, `status="filled"`, `quantity=0`, `price=0`) |
| Add stock to watchlist | `watchlist` array gains the symbol string |
| Remove/toggle stock from watchlist | `watchlist` array loses the symbol string |
| Mark single alert as read | `alerts[i].read` → `true` |
| Mark all alerts as read | All `alerts[i].read` → `true` |
| Update setting (dividend reinvestment) | `settings.dividendReinvestment` toggles boolean |
| Update setting (order notifications) | `settings.orderNotifications` toggles boolean |
| Update setting (theme) | `settings.theme` cycles through `"dark"` → `"light"` → `"auto"` |
| Price simulation tick (auto, every 3s) | All `stocks[i].currentPrice`, `stocks[i].change`, `stocks[i].changePercent` updated; `user.portfolioValue` recalculated |

**Note**: The `/crypto` page uses hardcoded constants (`CRYPTO_ASSETS`) and does NOT modify any state fields. Viewing the page causes no observable state change.

## Notes

- **Live price simulation**: Stock prices update every 3 seconds with random micro-fluctuations (±0.2% per tick). The `user.portfolioValue` is recalculated on every tick based on current holdings and live prices.
- **`history` field**: Auto-generated with ~101 daily price points when not provided in injected state. Each point: `{date: "YYYY-MM-DD", price: number, volume: number}`. Used for 1W, 1M, 3M, 1Y, 5Y, ALL chart ranges.
- **`intradayData` field**: Auto-generated with 30-minute price intervals from 9:30 AM to current time or 4:00 PM market close. Each point: `{date: "H:MM AM/PM", price: number, volume: number}`. Final point is scaled to equal `currentPrice`. Used exclusively for the 1D chart range on stock detail and portfolio dashboard.
- **Composite portfolio chart**: The Dashboard and Portfolio pages compute a weighted composite of all holdings' `history` (multi-day) or `intradayData` (1D), plus flat `cashBalance` contribution, to generate the portfolio value chart.
- **Transfer transactions**: `type="transfer"` entries appear in the History page when the "Transfer" type filter is selected. The Transfers page (`/transfers`) shows only `type="transfer"` entries in its "Recent Transfers" list.
- **News article modal**: Clicking any news item on Dashboard or StockDetail opens an in-page modal displaying headline, summary, image, and related symbol links. It does NOT navigate to an external URL or a new route.
- **State normalization on inject**: When custom state is injected via `/post`, the `deepMergeWithDefaults` function normalizes each stock through `normalizeStock()` (auto-generates `history` and `intradayData` if absent), each news item through `normalizeNews()`, each transaction through `normalizeTransaction()`. The `user` object is shallow-merged with defaults.
- **Session isolation**: Multiple `sid` values can run concurrently with independent state files and localStorage keys. This is used for parallel RL training episodes.
- **Order form (Dollars mode)**: When placing a buy/sell order using "Dollars" input mode on StockDetail, the dollar amount is converted to fractional shares (`quantity = dollars / price`). Market orders in "Shares" mode require whole numbers.
